# v3.6.0 Detection Architecture Migration Map

## Purpose

This document defines the target package structure for the Threat Hunting
Toolkit detection layer.

The v3.6.0 release changes module locations and import paths without changing
detection logic, findings, risk scores or reporting output.

---

## Refactoring Rules

- No detector behavior changes
- No new detection functionality
- Existing public function names remain unchanged
- Existing test expectations remain unchanged
- All moves must use `git mv`
- Imports are updated immediately after each migration group
- Full regression testing is required after every migration stage

---

src/detection/

## Components That Remain Outside Detection

The following packages already have clear architectural responsibilities and
will not be moved into `src/detection`:

```text
src/correlation/
src/intelligence/
src/mitre/
src/models/
src/parsers/
src/reporting/
src/utils/
```

Correlation, intelligence enrichment, models, parsers and reporting remain
separate platform layers.

---

## Migration Order

Files will be migrated in controlled groups:

1. Package skeleton
2. Nmap detectors
3. Firewall detector
4. Web detector
5. Network detectors
6. Windows detectors
7. Linux detectors
8. Application imports
9. Test imports
10. Cache cleanup
11. Full regression
12. Runtime validation
